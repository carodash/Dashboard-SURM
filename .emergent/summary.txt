The primary goal is to build a stable and reliable V1 of a Start-Up Relationship Management (SURM) dashboard to replace an inefficient Excel-based workflow. The application must centralize data for Sourcing and Dealflow management, automate workflows, and provide analytics.

**Key features implemented and requested:**
*   **Core Functionality:** A visual Kanban pipeline for deal flow, tables for Sourcing and Dealflow management, and a robust user permission system (Admin, Contributor, Observer).
*   **Document Management:** Ability to upload, download, version, and categorize various file types (.pdf, .docx, etc.) associated with each startup.
*   **Data Quality & Efficiency:**
    *   **Duplicate Detection:** An alert system that triggers during partner creation to check for existing similar entries, with options to view the existing record or force creation.
    *   **Data Enrichment:** An Enrich button to automatically populate form fields (domain, country, description) by pulling data from external sources like LinkedIn and Crunchbase, using free APIs.
*   **Advanced Data Interaction:**
    *   **Excel-like Filters:** Column-specific filters in Sourcing/Dealflow tables with multi-select checkboxes and sorting (alphabetical, date, numerical).
    *   **Formatted CSV Exports:** Ability to export all or selected data into a clean, human-readable CSV format with well-defined columns.
*   **Bug Fixes (High Priority):** Throughout the process, the user prioritized fixing critical bugs over new features, including non-functional buttons, drag-and-drop failures, and incorrect data displays.

**Explicitly Deprioritized for V1:**
*   Mobile optimization.
*   Future enhancements like fully customizable fields and a color-editable tag system.
